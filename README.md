# ChronoLogic — Інструкція з запуску

Застосунок для обробки технологічних операцій, призначення робітників та розрахунку часу виготовлення.

---

## Вимоги

| Інструмент | Версія        |
|------------|---------------|
| Python     | 3.10+         |
| Node.js    | 20.19+ або 22+|
| npm        | 9+            |

---

## Крок 1 — Запуск бекенду (FastAPI)

Відкрийте термінал і перейдіть до папки `app`:

```bash
cd app
```

### Створіть та активуйте віртуальне середовище

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### Встановіть залежності

```bash
pip install -r requirements.txt
```

### Запустіть сервер

```bash
python run.py
```

Бекенд буде доступний за адресою: **http://127.0.0.1:8000**

> Перевірити роботу: відкрийте http://127.0.0.1:8000/health — має повернути `{"status":"ok"}`

## Крок 2 — Запуск фронтенду (Vue 3 + Vite)

Відкрийте **другий термінал** і перейдіть до папки `app/frontend`:

```bash
cd app/frontend
```


```bash
npm run dev
```

Фронтенд буде доступний за адресою: **http://localhost:5173**

## Крок 3 — Робота з застосунком

### 3.1 Профілі робітників (необов'язково, але рекомендовано)

Натисніть кнопку **👷 Профілі** у правому верхньому куті.

На сторінці профілів ви можете:
- Додати робітника: ім'я, **розряд** (1–8), **тип обладнання**, **кількість одиниць обладнання**
- Редагувати або видалити будь-якого робітника
- Переглянути зведену статистику (загальна кількість обладнання, розбивка по розрядах)

> Якщо профілі заповнені, система автоматично призначить робітників з CSV-файлів відповідно до розряду та типу обладнання.

### 3.2 Завантаження CSV файлів

На головній сторінці:

1. Натисніть **📂 Обрати файли** — можна вибрати один або кілька CSV одночасно
2. Встановіть **кількість зразків** (кнопки −/+) — час буде помножено на це число
3. Натисніть **▶ Обробити**

### 3.3 Формат CSV

CSV-файл повинен містити **обов'язкові колонки**:

| Колонка                           | Опис                        |
|-----------------------------------|-----------------------------|
| `Затрати часу, хв`                | Час операції у хвилинах     |
| `Розряд`                          | Кваліфікаційний розряд (ціле число) |

**Необов'язкові колонки** (якщо присутні — враховуються):

| Колонка                          | Опис                            |
|----------------------------------|---------------------------------|
| `Блок`                           | Назва блоку / групи             |
| `Робітник`                       | Ім'я виконавця                  |
| `Обладнання`                     | Тип обладнання                  |
| `№ п/п`                          | Порядковий номер                |
| `№ тех.оп.`                      | Номер технологічної операції    |
| `Назва технологічної операції`   | Опис операції                   |
| `Технічні умови`                 | Технічні умови виконання        |

> Підтримується кодування: **UTF-8**, **UTF-8 BOM**, **Windows-1251 (CP1251)**, Latin-1.

### 3.4 Результати та розрахунок часу

Після обробки з'являється панель результатів:
- Кількість операцій, файлів, зразків
- Час обробки сервером
- **Таблиця часу по виконавцях**: загальний час (хв і год), час на 1 зразок

Натисніть кнопку **⏱ Час** поруч із виконавцем у таблиці для детальної інформації.

### 3.5 Редагування та експорт

- Усі поля таблиці редагуються прямо на сторінці (клік → введення)
- **+ Рядок** — додати порожній рядок вручну
- **↓ Експорт CSV** — зберегти поточну таблицю як CSV з BOM (UTF-8, сумісний з Excel)

## API ендпоінти

| Метод | Шлях                  | Опис                                               |
|-------|-----------------------|----------------------------------------------------|
| GET   | `/health`             | Перевірка стану сервера                            |
| POST  | `/api/process-fixed`  | Обробка одного CSV (без профілів)                  |
| POST  | `/api/process-multi`  | Обробка кількох CSV з профілями та кількістю зразків |

### POST `/api/process-multi` — параметри (multipart/form-data)

| Поле              | Тип       | Опис                                            |
|-------------------|-----------|-------------------------------------------------|
| `files`           | File[]    | Один або кілька CSV файлів                      |
| `workers_profile` | string    | JSON з профілями робітників (необов'язково)     |
| `sample_quantity` | integer   | Кількість зразків, за замовчуванням `1`         |

**Приклад `workers_profile`:**
```json
{
  "workers": [
    {
      "name": "Іванов І.І.",
      "rank": 4,
      "equipment_type": "Токарний верстат",
      "equipment_quantity": 2
    }
  ]
}
```

## Вирішення типових проблем

**`ModuleNotFoundError: No module named 'src'`**
— Запускайте бекенд лише з папки `app/`, а не з кореня проєкту.

**CORS помилка у браузері**
— Переконайтесь, що бекенд запущений на `http://127.0.0.1:8000`, а фронтенд на порту `5173` або `5174`.

**CSV не читається / помилка кодування**
— Збережіть файл як UTF-8 або Windows-1251 (CP1251) у Excel: `Зберегти як → CSV UTF-8`.

**`npm install` падає через версію Node.js`**
— Перевірте версію: `node -v`. Потрібна 20.19+ або 22+.
